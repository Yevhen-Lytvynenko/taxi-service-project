# Узгодження стеку реалізації Strum із текстом записки

Цей файл коротко зіставляє **що реально реалізовано в репозиторії** з типовими очікуваннями з академічної записки (карти, інфраструктура, експорт даних, ролі). Його мета — уникнути формулювань у роботі, які суперечать коду.

## Клієнтські застосунки

| Тема записки (можливий акцент) | Факт у проєкті |
|-------------------------------|----------------|
| Нативні карти Google Maps | Мобільні застосунки на **Expo / React Native**; карти через **OSM + Leaflet** (`OSMLeafletMap`), без офіційного Maps SDK у клієнті. |
| Єдиний стек з веб-адмінкою | Клієнт і водій — **React Native**; адмін-панель — **React + Vite + MUI**; спільний лише REST/WebSocket контракт з API. |

## Бекенд і дані

| Тема | Факт |
|------|------|
| PostGIS / просторові типи в БД | У схемі Prisma використовуються **поля `Float` для координат**, без geometry-типів PostgreSQL. |
| Черги повідомлений (RabbitMQ, MQTT тощо) | Розсилка подій водіям/кімнатам — **Socket.IO**; за потреби масштабування — опційний **Redis adapter** для Socket.IO. |
| Matching «як у проді» | Підбір водія спрощений (онлайн-профілі, виключення заблокованих тощо); окремі алгоритми записки можуть описувати як **цільову модель**, якщо вони ще не відображені 1:1 у коді. |

## Експорт і звітність

| Тема | Факт |
|------|------|
| Серверний PDF/Excel для всіх звітів | Частина аналітики в адмінці передбачає **експорт на клієнті** (CSV тощо у браузері); повний серверний пайплайн експорту не є обов’язковим для всіх розділів. |
| Роль бухгалтера | У системі є роль **`ACCOUNTANT`** у моделі користувача; доступ до операційних і фінансових endpoint’ів та пунктів меню налаштовується окремо від диспетчера. |

## Що варто явно прописати в записці

1. **Стек мобільних додатків**: Expo, React Native Paper, OSM/Leaflet замість вбудованих карт Google.
2. **Комунікація в реальному часі**: WebSocket (Socket.IO), не MQTT.
3. **Геодані**: зберігання широти/довготи у звичайних полях; геокодування через внутрішній сервіс геокодування API.
4. **RBAC**: перелік ролей узгодити з `Role` у Prisma та з обмеженнями маршрутів (`authorize.middleware`).

Оновлюйте цей файл, коли з’являються нові модулі (наприклад, повноцінний серверний експорт або PostGIS).
