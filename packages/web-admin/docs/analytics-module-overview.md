# Модуль аналітики в адмін-панелі (демонстраційний)

## Навіщо він

Операційна та стратегічна аналітика для диспетчерської таксі: **попит у часі**, **динаміка цін (surge)**, **географія замовлень**, **ефективність водіїв/автопарку**, **фінансові підсумки**. Це демо-шар поверх існуючих даних у БД (замовлення, треки, тарифи, транзакції тощо).

## Що вже показує UI (розділи меню)

| Розділ | Призначення |
|--------|-------------|
| **Прогноз і піки** | Годинні ряди замовлень, виявлення піків, простий прогноз наступних годин. |
| **Surge / ціноутворення** | Часові ряди коефіцієнтів/націнки (де реалізовано на бекенді). |
| **Гео та теплокарти** | Просторова щільність подій (підбір/трафік по сітці або hex). |
| **Водії та автопарк** | KPI водіїв: завантаженість, завершені поїздки, ефективність маршрутів (за наявності даних). |
| **Фінанси та звіти** | Денні/періодні агрегати виручки, витрат, експорт для бухгалтерії. |

## Які метрики варто «захоплювати» (цільова модель)

Нижче — орієнтир для продукту, не вимога до поточного демо.

### Попит і пропозиція

- Замовлення за годину / день / зону (створені, прийняті, скасовані, завершені).
- Час до призначення водія (matching latency), частка відмов.
- Активні водії онлайн vs кількість замовлень (баланс ринку).

### Ціноутворення

- Базова ціна vs фактична (surge, знижки, промо).
- Середній чек, розподіл по тарифах/класах авто.

### Гео

- Теплокарти подачі та висадки (H3/сітка), «гарячі» райони за період.
- Середня довжина/тривалість поїздки по зонах.

### Флот

- Використання авто (години «на лінії», поїздки на зміну).
- Рейтинги, скарги, completion rate по водію.

### Фінанси

- GMV, комісія платформи, виплати водіям, податкові/операційні витрати (OPEX).
- ARPU, LTV клієнта (якщо є облікові профілі та повторні поїздки).

### Якість сервісу

- NPS/оцінки після поїздки, причини скасувань.

## Технічні примітки

- Джерело даних: PostgreSQL + агрегуючі запити (`packages/backend/src/services/analytics.service.ts`).
- Період у UI задається спільно для всіх сторінок розділу (фільтр дат у шапці layout).
- Для бойового використання додають кеш (Redis), матеріалізовані представлення, черги для важких звітів та RBAC по ролях.
