# Інструкція для демонстрації такси-системи (захист диплома)

## Підготовка

### 1. Запуск інфраструктури

```bash
# PostgreSQL
docker-compose up -d

# Seed (20 клієнтів, 50 водіїв, демо-акаунти)
cd packages/backend && npx prisma db push && npx prisma db seed

# Backend API + Socket.IO (з кореня проекту)
npm run backend

# (опційно) Окремий сервіс демо-GPS — автозамовлення, рух по OSRM (не частина backend)
npm run gps-sim
# або: npm run emulation (аліас на gps-sim)
```

### 2. Запуск додатків (з кореня проекту)

- **web-admin**: `npm run admin`
- **app-client**: `npm run client` (Expo)
- **app-driver**: `npm run driver` (Expo)

**Мобільні додатки (Expo):**

- Android **емулятор**: за замовчуванням використовується `http://10.0.2.2:4000/api` (доступ до backend на ПК).
- iOS **симулятор**: `http://localhost:4000/api`.
- **Фізичний телефон** у Wi‑Fi: скопіюйте `packages/app-client/.env.example` у `packages/app-client/.env` (те саме для `app-driver`), вкажіть `EXPO_PUBLIC_API_URL=http://ВАШ_IP:4000/api`, перезапустіть Expo.
- У `app.json` увімкнено `usesCleartextTraffic` для HTTP.

## Сценарій 1: Ручна демо (ви самі робите заказ)

1. **app-client**: увійти як демо-клієнт (`+380991111111` / `demo`)
2. Створити замовлення: обрати точки підбору та висадки на карті або ввести адреси
3. **app-driver**: увійти як демо-водій (`demodriver` / `demo`)
4. Перейти в режим ONLINE (якщо ще не онлайн)
5. Прийняти замовлення з черги
6. Натиснути **«Симулювати поїздку»** — запуститься автоматична симуляція руху по маршруту (OSRM)
7. Спостерігати, як авто рухається по карті, статуси змінюються (ACCEPTED → ARRIVED → IN_PROGRESS → COMPLETED)

---

## Сценарій 2: Автоматична емуляція

Якщо запущено **`npm run gps-sim`** (пакет `@taxi/gps-simulator`):

- Процес **не** імпортує Prisma/backend: тільки HTTP (`/api/emulation/*`) + Socket.IO `update_location`, як майбутній IoT/телефон
- 50 водіїв рухаються в межах Одеси по геометрії **OSRM**
- Кожні 45–90 секунд створюється нове замовлення через API; призначається найближчий вільний водій
- Статуси замовлення оновлюються через захищені HTTP-ендпоінти емуляції
- У web-admin Dashboard відображається дохід від завершених замовлень

---

## Що показує система

- **web-admin**: замовлення, водії, дохід, моніторинг в реальному часі
- **app-client**: створення замовлення, відстеження поїздки
- **app-driver**: черга замовлень, прийняття, симуляція поїздки, завершення
- **Socket.IO**: оновлення позицій водіїв і статусів замовлень в реальному часі

---

## Troubleshooting

- Якщо емуляція не створює замовлення — перевірте, що seed виконано і є клієнти/водії
- Якщо «Симулювати поїздку» не працює — backend має бути запущений, водій має бути призначений на замовлення
- Для Expo: налаштуйте `API_URL` в app-client та app-driver під вашу мережу (наприклад, `http://192.168.x.x:4000/api`)
